# Caramel Web - Project Overview

Caramel Web is the landing page and official website for **Caramel**, a modular Discord bot built with TypeScript. The project is designed to showcase the bot's features, tech stack, and roadmap through a highly interactive and visually polished user interface.

## 🛠️ Tech Stack

- **Framework:** [Next.js 16 (App Router)](https://nextjs.org/)
- **Library:** [React 19](https://react.dev/)
- **Language:** [TypeScript](https://www.typescriptlang.org/)
- **Styling:** [Tailwind CSS 4](https://tailwindcss.com/)
- **Animations:** [Framer Motion](https://www.framer.com/motion/), [GSAP](https://gsap.com/), and [OGL](https://github.com/o-g-l/ogl) (for advanced shader effects like the Aurora background).
- **Icons:** [Phosphor Icons](https://phosphoricons.com/)
- **UI Components:** [Shadcn UI](https://ui.shadcn.com/) patterns with custom interactive components.
- **Internationalization:** Custom context-based i18n system supporting English (`en`) and Spanish (`es`).

## 📁 Key Directories & Architecture

- `src/app/`: Contains the Next.js App Router structure, including the root layout and the main landing page.
- `src/components/ui/`: A rich library of reusable UI components, many focusing on animations (e.g., `Aurora`, `BlurText`, `SplitText`) and specialized simulations (e.g., `DiscordTerminal`).
- `src/lib/`:
    - `i18n-context.tsx`: Manages the application's language state.
    - `translations.ts`: Centralized dictionary for all text in the supported languages.
    - `utils.ts`: Standard utility functions (e.g., the `cn` helper for Tailwind class merging).
- `public/`: Assets including branding icons and specific images for the Discord terminal simulation.

## 🧩 Development Conventions

### 1. Internationalization (i18n)
All user-facing text must be added to `src/lib/translations.ts` in both English and Spanish. Access translations in components using the `useLanguage` hook:
```tsx
const { t } = useLanguage();
// Use t.hero.title, etc.
```

### 2. UI & Styling
- Use **Tailwind CSS 4** for styling.
- Utilize the `cn()` utility for conditional class merging.
- Prefer existing animation components in `src/components/ui` to maintain consistent motion design across the site.

### 3. TypeScript
- Maintain strict typing throughout the project.
- Define interfaces for complex data structures (e.g., the `Translation` interface in `translations.ts`).

### 4. Interactive Components
- The `DiscordTerminal` component is a key feature that simulates the Discord UI. When adding new bot command previews, update the `commands` array within `src/components/ui/DiscordTerminal.tsx`.
